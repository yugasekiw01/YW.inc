# エージェント連携システム 要件定義ドラフト

**ステータス**: Phase 1 - 要件定義中  
**作成者**: David  
**作成日**: 2026-05-18

---

## 1. 機能の目的

### 誰が
- YW.inc のユーザー（スマホ/PC）
- 各エージェント（David, Tiara, Imai, Conner, Sakura）

### 何を
- エージェント間のタスク受け渡しを自動化
- 各エージェントの作業状況をリアルタイムで可視化

### なぜ
- 現状: 手動でエージェントにタスクを伝える必要がある
- 目標: W字モデルに沿って、フェーズ完了時に次のエージェントへ自動でタスクが流れる
- 効果: スマホからでもエージェントの進捗が見え、手戻りが減る

## 3. ユースケース

### UC-1: ユーザーがタスクを依頼
```
1. ユーザーがダッシュボードで「Tiaraにテストを依頼」
2. Supabaseにタスクが登録される
3. Tiara がタスクを受け取り実行
4. 進捗がリアルタイムでダッシュボードに反映
```

### UC-2: エージェントが次のエージェントに自動引き継ぎ（W字モデル全パターン）

**W字モデルのフェーズ遷移:**
```
Phase 1: 要件定義  → Phase 2: 実装
  Conner ─────────→ David（仕様書を渡す）
  Tiara  ─────────→ David（テスト仕様を渡す）
  Imai   ─────────→ David（インフラ要件を渡す）

Phase 2: 実装 → Phase 3: テスト
  David ──────────→ Tiara（テスト依頼）

Phase 3: テスト → Phase 2: 実装（バグ発見時）
  Tiara ──────────→ David（バグ修正依頼）

Phase 3: テスト → Phase 4: リリース
  Tiara ──────────→ Imai（デプロイ依頼）

Phase 4: リリース → Phase 5: 振り返り
  Imai ───────────→ Conner（振り返り依頼）

Phase 5: 振り返り → 次スプリント or 完了
  Conner ─────────→ Sakura（次スプリント開始 or 完了報告）
```

**フロー例（David→Tiara）:**
```
1. David が実装完了
2. David が「テスト依頼」タスクをTiara宛に作成
3. システムがTiaraを自動起動
4. Tiara がタスクを受け取り実行開始
5. ダッシュボードのフェーズが「実装」→「テスト」に自動遷移
```

**フロー例（Tiara→David バグ発見時）:**
```
1. Tiara がテスト中にバグ発見
2. Tiara が「バグ修正依頼」タスクをDavid宛に作成
3. システムがDavidを自動起動
4. フェーズが「テスト」→「実装」に戻る（フィードバックループ）
```

### UC-3: スマホから進捗確認
```
1. ユーザーがスマホでダッシュボードを開く
2. 各エージェントの現在のタスク・進捗が見える
3. W字モデルのどのフェーズにいるかが一目でわかる
```

## 4. 受け入れ条件（完成の定義）

### 必須
- [ ] タスクを作成できる
- [ ] タスクのステータス（pending/running/done/failed）が更新できる
- [ ] ダッシュボードで全エージェントの状態が見える
- [ ] スマホでも正常に表示される（レスポンシブ）
- [ ] Supabase Realtimeで自動更新される

### 推奨
- [ ] W字モデルのフェーズ表示（要件定義/実装/テスト/リリース/振り返り）
- [ ] タスク履歴が見える
- [ ] エージェントのログ/出力の一部が見える

## 5. データ設計（案）

### テーブル: `agent_tasks`
| カラム | 型 | 説明 |
|--------|-----|------|
| id | uuid | PK |
| sprint_name | text | スプリント名（例: kaidashi-v1） |
| phase | text | W字モデルのフェーズ |
| from_agent | text | 依頼元エージェント |
| to_agent | text | 依頼先エージェント |
| task_type | text | タスク種別（dev/test/deploy等）|
| title | text | タスクのタイトル |
| description | text | 詳細説明 |
| status | text | pending/running/done/failed |
| progress | int | 進捗% (0-100) |
| output | text | 実行結果/ログ |
| created_at | timestamp | 作成日時 |
| updated_at | timestamp | 更新日時 |

### テーブル: `agent_status`
| カラム | 型 | 説明 |
|--------|-----|------|
| agent_name | text | PK |
| status | text | idle/busy/offline |
| current_task_id | uuid | 現在実行中のタスク |
| last_seen | timestamp | 最終アクティブ時刻 |
