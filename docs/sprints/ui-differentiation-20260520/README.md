# スプリント：カイダシ UI 差別化

**開始日**: 2026-05-20  
**ステータス**: 要件定義フェーズ  
**優先度**: 高（法的リスク対応）

---

## 目的

カイダシのイベント作成フローが既存サービス「walica」と酷似しており、著作権・商標権侵害のリスクがある。
UIフローを変更し、カイダシ独自の価値を前面に出すことで差別化を図る。

---

## 背景

- **現状**: トップページでイベント名を入力 → 作成ボタン → 成功モーダル → URL共有
- **問題**: walica と同様のフローで、UIの模倣と見られるリスク
- **対応方針**: 買い出しリスト機能を前面に出し、割り勘は補助機能として位置づける
